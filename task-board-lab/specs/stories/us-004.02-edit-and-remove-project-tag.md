# User Story: Edit and Remove Project Tag

**Story ID:** US-004.02
**Epic:** EPIC-004 — Project Focus and Board Maintenance Controls
**Author:** GitHub Copilot
**Status:** Backlog
**Sprint:** Unassigned
**Last Updated:** 2026-05-05

---

## 1. User Story

> As a Maya,
> I want to edit or remove a task's project tag,
> so that I can keep project grouping accurate when priorities change.

---

## 2. Acceptance Criteria

- [ ] **AC-1:** Given a task already has a project tag, when I edit the task and change the tag, then the updated tag is shown on the task after save.
- [ ] **AC-2:** Given a task has a project tag, when I clear the tag field and save, then the task is saved with no project tag.
- [ ] **AC-3:** Given I cancel editing a task, when I return to board view, then the task tag remains unchanged.
- [ ] **AC-4:** Given I update a tag value, when I refresh the page, then the edited value persists.

---

## 3. Technical Notes

- **Affected components:** TaskEditModal.tsx [Assumed], TaskCard.tsx, useLocalStorage.ts
- **API / Data considerations:** Reuse the same optional projectTag field introduced in task model and ensure immutable state update for edited tasks.
- **Edge cases to watch:** Editing while a filter is active, removing a tag used by other tasks, rapid consecutive edits.
- **Security / compliance notes:** Avoid writing partial edits until save is confirmed.
- **External dependencies:** Depends on basic task edit flow from EPIC-001.
