# User Story: Add Project Tag During Task Creation

**Story ID:** US-004.01
**Epic:** EPIC-004 — Project Focus and Board Maintenance Controls
**Author:** GitHub Copilot
**Status:** Done
**Sprint:** Sprint 1-2 (Foundation + EPIC-004)
**Last Updated:** 2026-05-07
**Implemented By:** GitHub Copilot (v0.1.0)

---

## 1. User Story

> As a Ethan,
> I want to assign a project tag when creating a task,
> so that I can immediately organize new work under the right project context.

---

## 2. Acceptance Criteria

- [x] **AC-1:** Given I open the new task form, when the form is displayed, then I can enter an optional project tag value before saving.
  - ✅ Implemented: TaskForm component includes `projectTag` input field (line 67-77 in `src/components/TaskForm.tsx`)
- [x] **AC-2:** Given I provide a task title and a project tag, when I save the task, then the created task appears in the board with the same tag value.
  - ✅ Implemented: projectTag preserved in task creation (line 42-45 in `src/components/TaskBoard.tsx`), displayed as blue badge on TaskCard (line 30-33 in `src/components/TaskCard.tsx`)
- [x] **AC-3:** Given I create a task without entering a project tag, when the task is saved, then the task is created successfully with no tag assigned.
  - ✅ Implemented: projectTag is optional; trimProjectTag returns undefined for empty strings (line 22-27 in `src/utils/helpers.ts`)
- [x] **AC-4:** Given I enter leading or trailing spaces in a project tag, when I save the task, then the stored tag is trimmed.
  - ✅ Implemented: trimProjectTag utility handles all whitespace cases (line 22-27 in `src/utils/helpers.ts`), called on form submit (line 75 in `src/components/TaskForm.tsx`)

---

## 3. Technical Notes

- **Affected components:** TaskForm.tsx [Assumed], TaskCard.tsx, useLocalStorage.ts
- **API / Data considerations:** Extend the task type with optional projectTag string and include it in serialized localStorage state.
- **Edge cases to watch:** Empty tag input, whitespace-only input, tags containing mixed case.
- **Security / compliance notes:** Keep all tag handling local to browser storage and do not transmit data externally.
- **External dependencies:** None.
