# User Story: Restore Last Project Filter Preference

**Story ID:** US-004.05
**Epic:** EPIC-004 — Project Focus and Board Maintenance Controls
**Author:** GitHub Copilot
**Status:** Backlog
**Sprint:** Unassigned
**Last Updated:** 2026-05-05

---

## 1. User Story

> As a Ethan,
> I want my last selected project filter restored on reload,
> so that I can continue where I left off without reapplying context.

---

## 2. Acceptance Criteria

- [ ] **AC-1:** Given I select a project filter and refresh the page, when the board reloads, then the same filter is automatically applied.
- [ ] **AC-2:** Given no filter was selected in the previous session, when the board reloads, then all tasks are shown by default.
- [ ] **AC-3:** Given the saved filter tag no longer exists on any task, when the board reloads, then the app falls back to full-board view and clears invalid saved filter state.
- [ ] **AC-4:** Given I clear the active filter and refresh the page, when the board reloads, then no filter is active.

---

## 3. Technical Notes

- **Affected components:** useLocalStorage.ts, TaskBoard.tsx [Assumed], ProjectFilterBar.tsx [Assumed]
- **API / Data considerations:** Store selected project filter key separately from task array to avoid coupling filter state to task serialization changes.
- **Edge cases to watch:** Stale filter key, first app load with empty storage, localStorage read failures.
- **Security / compliance notes:** Persist only non-sensitive filter strings.
- **External dependencies:** Depends on filter selection behavior from US-004.03.
