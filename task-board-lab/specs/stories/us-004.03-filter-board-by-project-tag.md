# User Story: Filter Board by Project Tag

**Story ID:** US-004.03
**Epic:** EPIC-004 — Project Focus and Board Maintenance Controls
**Author:** GitHub Copilot
**Status:** Backlog
**Sprint:** Unassigned
**Last Updated:** 2026-05-05

---

## 1. User Story

> As a Ethan,
> I want to filter the board by one project tag,
> so that I can focus on one project's tasks without scanning unrelated work.

---

## 2. Acceptance Criteria

- [ ] **AC-1:** Given tasks have at least two different project tags, when I select one tag in the filter control, then only tasks with that tag are displayed across all columns.
- [ ] **AC-2:** Given a project filter is active, when I move a visible task between columns, then the task remains visible if it still matches the active tag.
- [ ] **AC-3:** Given a project filter is active, when I create a task with a different tag, then the new task is hidden from the filtered view.
- [ ] **AC-4:** Given no tasks match the selected tag, when the filter is applied, then the board shows a clear empty-state message for that filtered view.

---

## 3. Technical Notes

- **Affected components:** TaskBoard.tsx [Assumed], ProjectFilterBar.tsx [Assumed], TaskColumn.tsx [Assumed]
- **API / Data considerations:** Apply filter as a derived selector over shared task state rather than mutating underlying task collections.
- **Edge cases to watch:** Untagged tasks under active filter, case sensitivity of tag matching, filter behavior during drag operations.
- **Security / compliance notes:** None beyond existing local-only data constraints.
- **External dependencies:** Depends on project tag field availability in task model.
