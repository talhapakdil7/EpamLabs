# User Story: Clear Board with Confirmation Guard

**Story ID:** US-004.06
**Epic:** EPIC-004 — Project Focus and Board Maintenance Controls
**Author:** GitHub Copilot
**Status:** Backlog
**Sprint:** Unassigned
**Last Updated:** 2026-05-05

---

## 1. User Story

> As a Maya,
> I want a confirmation step before clearing the board,
> so that I do not accidentally delete all tasks.

---

## 2. Acceptance Criteria

- [ ] **AC-1:** Given tasks exist on the board, when I choose Clear Board, then a confirmation prompt appears with Cancel and Confirm actions.
- [ ] **AC-2:** Given the confirmation prompt is open, when I select Cancel, then no tasks are deleted and the board remains unchanged.
- [ ] **AC-3:** Given the confirmation prompt is open, when I select Confirm, then all tasks are removed from all columns and the board shows the empty-board state.
- [ ] **AC-4:** Given I confirm clear board and refresh the page, when the app reloads, then no previously deleted tasks reappear.
- [ ] **AC-5:** Given a clear-board action succeeds, when the prompt closes, then a visible success feedback message is shown.

---

## 3. Technical Notes

- **Affected components:** BoardSettingsMenu.tsx [Assumed], ConfirmDialog.tsx [Assumed], useLocalStorage.ts
- **API / Data considerations:** Clear both in-memory task state and persisted task keys in localStorage atomically within one user action.
- **Edge cases to watch:** Double-submit on confirm button, prompt dismissal via Escape key, clearing while a filter is active.
- **Security / compliance notes:** None.
- **External dependencies:** Depends on baseline board state management from EPIC-001.
